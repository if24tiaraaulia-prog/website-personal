Post 1 — Tata cara instalasi 1 DBMS (MySQL)

Di bawah ini ada video tutorial (YouTube) yang sudah disematkan, lalu panduan langkah demi langkah singkat untuk instalasi MySQL pada sistem operasi Windows dan Linux (Ubuntu).

Ringkasan langkah instalasi (Windows)

  1. Download MySQL Installer dari situs resmi MySQL (pilih MySQL Installer for Windows).
  2. Jalankan installer → pilih "Developer Default" atau "Server only" sesuai kebutuhan.
  3. Ikuti wizard: pilih fitur (MySQL Server, Workbench, Shell), lalu Next → Install.
  4. Set root password saat diminta. Catat username (root) dan password.
  5. Set konfigurasi port (default 3306) dan pilih opsi Authentication (use strong password recommended).
  6. Selesai — buka MySQL Workbench atau MySQL Shell, lalu koneksi ke server menggunakan root:3306.

Langkah singkat instalasi (Ubuntu / Debian)

Jalankan perintah berikut di terminal (contoh untuk MySQL 8.x):

sudo apt update
sudo apt install mysql-server
# secure installation (mengatur root password & opsi keamanan)
sudo mysql_secure_installation
# jalankan mysql shell
sudo mysql -u root -p

Contoh membuat database dan user

-- login ke mysql lalu jalankan:
CREATE DATABASE toko;
CREATE USER 'admin'@'localhost' IDENTIFIED BY 'Passw0rd!';
GRANT ALL PRIVILEGES ON toko.* TO 'admin'@'localhost';
FLUSH PRIVILEGES;
-- verify
SHOW DATABASES;

Post 2 — Apa itu atribut, entitas dan relasi dalam ERD?

ERD (Entity-Relationship Diagram) adalah diagram yang digunakan untuk memodelkan struktur data dan hubungan antar entitas dalam sebuah sistem informasi. Berikut penjelasan dasar beserta contoh.

Apa itu Entitas?

Entitas adalah objek nyata atau konsep yang datanya ingin disimpan. Contoh: Mahasiswa, Dosen, MataKuliah, Pesanan.

Apa itu Atribut?

Atribut adalah informasi yang menggambarkan entitas. Contoh atribut untuk entitas Mahasiswa:

  • nim (primary key)
  • nama
  • tanggal_lahir
  • email

Apa itu Relasi (Relationship)?

Relasi menunjukkan hubungan antara dua (atau lebih) entitas. Contoh: Mahasiswa mengambil MataKuliah. Relasi dapat memiliki kardinalitas:

  • 1:1 (one-to-one)
  • 1:N (one-to-many)
  • M:N (many-to-many) — biasa dipecah menjadi tabel/junction table.

Contoh ERD sederhana (Mahasiswa — MataKuliah)

Contoh tabel (relasi 1:N)

Skema tabel sederhana:

-- tabel mahasiswa
CREATE TABLE mahasiswa (
  nim VARCHAR(12) PRIMARY KEY,
  nama VARCHAR(100),
  email VARCHAR(100)
);

-- tabel matakuliah
CREATE TABLE matakuliah (
  kode_mk VARCHAR(10) PRIMARY KEY,
  nama_mk VARCHAR(120),
  sks INT
);

-- tabel penghubung (jika many-to-many) atau jika mahasiswa mengambil banyak MK,
-- gunakan tabel nilai / mengambil:
CREATE TABLE mengambil (
  id INT AUTO_INCREMENT PRIMARY KEY,
  nim VARCHAR(12),
  kode_mk VARCHAR(10),
  semester VARCHAR(10),
  nilai CHAR(2),
  FOREIGN KEY (nim) REFERENCES mahasiswa(nim),
  FOREIGN KEY (kode_mk) REFERENCES matakuliah(kode_mk)
);

Catatan: Jika hubungan M:N, buat tabel penghubung seperti mengambil di atas.
